# 常见集合

不同于内建的数组和元组类型，这些集合指向的数据是储存在堆上的

这意味着数据的数量不必在编译时就已知，并且还可以随着程序的运行增长或缩小

常见的三个集合:

- `vector` 允许我们一个挨着一个地储存一系列数量可变的值
- 字符串（`string`）是字符的集合
- 哈希 `map`（`hash map`）允许我们将值与一个特定的键（`key`）相关联。这是一个叫做 `map` 的更通用的数据结构的特定实现

不同于内建的数组和元组类型，这些集合指向的数据是储存在堆上的，这意味着数据的数量不必在编译时就已知，并且还可以随着程序的运行增长或缩小

## Vector

`vector` 允许我们在一个单独的数据结构中储存多个值，所有值在内存中彼此相邻排列

`vector` 只能储存相同类型的值

### 新建 vector

> 必须标注类型

1. 使用 `new` 函数：`let v: Vec<i32> = Vec::new();`

2. 使用 `vec!` 宏：`let v = vec![1, 2, 3];`

### 更新 vector

```rust
#![allow(unused)]
fn main() {
    let mut v = Vec::new();

    v.push(5);
    v.push(6);
    v.push(7);
    v.push(8);
}
```

### 丢弃 vector 时也会丢弃其所有元素

```rust

#![allow(unused)]
fn main() {
    {
        let v = vec![1, 2, 3, 4];

        // 处理变量 v

    } // <- 这里 v 离开作用域并被丢弃
}

```

### 读取 vector 的元素

有两种方式取得元素

```rust

#![allow(unused)]
fn main() {
    let v = vec![1, 2, 3, 4, 5];

    let third: &i32 = &v[2];
    println!("The third element is {}", third);

    // 这里取得的类型是 Option<&T>
    match v.get(2) {
        Some(third) => println!("The third element is {}", third),
        None => println!("There is no third element."),
    }
}

```

一旦程序获取了一个有效的引用，借用检查器将会执行所有权和借用规则来确保 `vector` 内容的这个引用和任何其他引用保持有效

```rust

#![allow(unused)]
fn main() {
    let mut v = vec![1, 2, 3, 4, 5];

    let first = &v[0];

    v.push(6);

    // 报错
    println!("The first element is: {}", first);
}

```

### 遍历 vector 中的元素

```rust

#![allow(unused)]
fn main() {
    // 不可变
    let v1 = vec![100, 32, 57];
    for i in &v1 {
        println!("{}", i);
    }

    // 可变
    let mut v2 = vec![100, 32, 57];
    for i in &mut v2 {
        // 解引用
        *i += 50;
    }
}

```

### 使用枚举来储存多种类型

`vector` 只能存储相同类型，此时可以利用枚举来创建多个不同的值，然后用一个 `vector` 来存储不同的枚举

```rust

#![allow(unused)]
fn main() {
    enum SpreadsheetCell {
        Int(i32),
        Float(f64),
        Text(String),
    }

    let row = vec![
        SpreadsheetCell::Int(3),
        SpreadsheetCell::Text(String::from("blue")),
        SpreadsheetCell::Float(10.12),
    ];
}

```

## String

字符串是比很多开发者所想象的要更为复杂的数据结构，以及 `UTF-8`

### 什么是字符串

```text
Rust 的核心语言中只有一种字符串类型：`str`，字符串 `slice`，它通常以被借用的形式出现，`&str`
第 4 章讲到了 字符串 `slice`：它们是一些储存在别处的 `UTF-8` 编码字符串数据的引用
比如字符串字面量被储存在程序的二进制输出中，字符串 `slice` 也是如此
```

### 新建字符串

很多 `Vec` 可用的操作在 `String` 中同样可用

```rust

#![allow(unused)]
fn main() {
    let mut s = String::new();

    let data = "initial contents";

    let s = data.to_string();

    // 该方法也可直接用于字符串字面量：
    let s1 = "initial contents".to_string();

    // 也可以使用from函数
    let s2 = String::from("initial contents");
}

```

`String::from` 和 `to_string` 最终做到了完全相同的事情，所以如何选择，就是风格问题了

### 更新字符串

#### 使用 push_str 和 push 附加字符串

按照 `vec` 那样使用即可

#### 使用 + 运算符或 format! 宏拼接字符串

```rust

#![allow(unused)]
fn main() {
    let s1 = String::from("Hello, ");
    let s2 = String::from("world!");
    let s3 = s1 + &s2; // 注意 s1 被移动了，不能继续使用
    // rustc 遇到+号之后执行的伪函数：fn add(self, s: &str) -> String {}
}

```

首先，`s2` 使用了 `&`，意味着我们使用第二个字符串的 引用 与第一个字符串相加。

这是因为 `add` 函数的 `s` 参数：只能将 `&str` 和 `String` 相加，不能将两个 `String` 值相加

这里一个类型是 `&str`，一个是 `String`，能相加的原因是 `&String` 可以被 强转（`coerced`）成 `&str`

因为 `add` 函数没有获取参数的所有权，所以 `s2` 在这个操作后仍然是有效的 `String`

虽然 `let s3 = s1 + &s2;` 看起来就像它会复制两个字符串并创建一个新的字符串，而实际上这个语句会获取 `s1` 的所有权，附加上从
`s2` 中拷贝的内容，
并返回结果的所有权；换句话说，它看起来好像生成了很多拷贝，不过实际上并没有：这个实现比拷贝要更高效。

对于更为复杂的字符串连接，可以使用 `format!` 宏

```rust

#![allow(unused)]
fn main() {
    let s1 = String::from("tic");
    let s2 = String::from("tac");
    let s3 = String::from("toe");

    let s = format!("{}-{}-{}", s1, s2, s3);
}

```

### 索引字符串

`rust` 中的字符串不支持使用索引访问，因为要处理不同文字的编码问题

#### 字符串 slice

索引字符串通常是一个坏点子，因为字符串索引应该返回的类型是不明确的：字节值、字符、字形簇或者字符串 `slice`

```rust

#![allow(unused)]
fn main() {
    let hello = "Здравствуйте";

    let s = &hello[0..4];
}

```

`&hello[0..1]`会panic，因为切片切出来的不是utf标准字符

### 遍历字符串的方法

char 方法:

```rust

#![allow(unused)]

    for c in "नमस्ते".chars() {
        println!("{}", c);
    }
}

```

bytes 方法:

```rust

    for b in "नमस्ते".bytes() {
        println!("{}", b);
    }
}

```

## Map

`HashMap<K, V>` 类型储存了一个键类型 `K` 对应一个值类型 `V` 的映射

### 新建一个哈希 map

可以使用 `new` 创建一个空的 `HashMap`，并使用 `insert` 增加元素

```rust

#![allow(unused)]
fn main() {
    // 这是必须的
    use std::collections::HashMap;

    let mut scores = HashMap::new();

    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);
}

```

像 `vector` 一样，哈希 `map` 将它们的数据储存在堆上

另一个构建哈希 `map` 的方法是使用一个元组的 `vector` 的 `collect` 方法，其中每个元组包含一个键值对

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let teams = vec![String::from("Blue"), String::from("Yellow")];
    let initial_scores = vec![10, 50];

    let scores: HashMap<_, _> = teams.iter().zip(initial_scores.iter()).collect();
}

```

### 哈希 map 和所有权

对于像 `i32` 这样的实现了 `Copy trait` 的类型，其值可以拷贝进哈希 `map`

对于像 `String` 这样拥有所有权的值，其值将被移动而哈希 `map` 会成为这些值的所有者

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let field_name = String::from("Favorite color");
    let field_value = String::from("Blue");

    let mut map = HashMap::new();
    map.insert(field_name, field_value);
    // 这里 field_name 和 field_value 不再有效，
    // 尝试使用它们看看会出现什么编译错误！
}

```

如果将值的引用插入哈希 `map`，这些值本身将不会被移动进哈希 `map`。但是这些引用指向的值必须至少在哈希 `map` 有效时也是有效的

### 访问哈希 map 中的值

1.使用 `get` 方法并提供对应的 `key` 来访问

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let mut scores = HashMap::new();

    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    let team_name = String::from("Blue");
    // 这里的类型是 Option<?>，值是Some()或None，可以使用match处理
    let score = scores.get(&team_name);
}

```

2.通过 `for` 访问

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let mut scores = HashMap::new();

    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    for (key, value) in &scores {
        println!("{}: {}", key, value);
    }
}

```

### 更新哈希 map

#### 覆盖一个值

重复对一个键插入两次，第二次的值就会覆盖第一次的

#### 只在键没有对应值时插入

我们经常会检查某个特定的键是否有值，如果没有就插入一个值

为此哈希 `map` 有一个特有的 `API`，叫做 `entry`，它获取我们想要检查的键作为参数

`entry` 函数的返回值是一个枚举，`Entry`，它代表了可能存在也可能不存在的值

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);

    // 这里是一个枚举，分别代表有值和没值
    // 如果 Yellow 没有值就插入，有就跳过
    scores.entry(String::from("Yellow")).or_insert(50);
    scores.entry(String::from("Blue")).or_insert(50);

    println!("{:?}", scores);
}

```

#### 根据旧值更新一个值

通过哈希 `map` 储存单词和计数来统计出现次数

```rust

#![allow(unused)]
fn main() {
    use std::collections::HashMap;

    let text = "hello world wonderful world";

    let mut map = HashMap::new();

    for word in text.split_whitespace() {
        let count = map.entry(word).or_insert(0);
        // 解引用
        *count += 1;
    }

    println!("{:?}", map);
}

```
