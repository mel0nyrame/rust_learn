# 错误处理

`Rust` 有很多特性来处理出现错误的情况

在很多情况下，`Rust` 要求你承认出错的可能性，并在编译代码之前就采取行动，确保将代码部署到生产环境之前就发现错误并正确地处理它们

错误类别：
  - 可恢复错误（recoverable）：比如未找到文件
  - 不可恢复错误（unrecoverable）：比如尝试访问超过数组结尾的位置

大部分语言并不区分这两类错误，并采用类似异常这样方式统一处理他们。

`Rust` 并没有异常，但是，有可恢复错误 `Result<T, E>` ，和不可恢复(遇到错误时停止程序执行)错误 `panic!`

## panic! 与不可恢复的错误

`Rust` 的 `panic` 宏类似于 `Java` 的 `Exception` 直接抛出，但是又有区别

当执行 `panic!` 的时候有两种情况：
  -  展开：Rust 会回溯栈并清理它遇到的每一个函数的数据，不过这个回溯并清理的过程有很多工作
  -  终止：这会不清理数据就退出程序，那么程序所使用的内存需要由操作系统来清理

如果想要项目的最终二进制文件越小越好，可以直接设置:

```toml
[profile.release]
panic = 'abort' # 设置panic宏为终止
```

示例代码：

```rust
fn main() {
    panic!("crash and burn");
}
```

结果：

```text
$ cargo run
   Compiling panic v0.1.0 (file:///projects/panic)
    Finished dev [unoptimized + debuginfo] target(s) in 0.25s
     Running `target/debug/panic`
thread 'main' panicked at 'crash and burn', src/main.rs:2:5
note: Run with `RUST_BACKTRACE=1` for a backtrace.
```

在其他情况下，`panic!` 可能会出现在我们的代码所调用的代码中，因此我们可以使用 `panic!` 被调用的函数的 `backtrace` 来寻找代码中出问题的地方
