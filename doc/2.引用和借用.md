# rust中的引用和借用

>和 `c/c++` 中的指针差不多,但是 rust 有很多安全检查

在变量前添加 `&` 符号表示**引用**,它们允许使用值但不获取其所有权,例如

```rust
fn main() {
    let s1 = String::from("hello");

    let len = calculate_length(&s1);

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

```

创建一个引用的行为称为**借用**

引用变量的可变性取决于原先变量的可变性

```rust
fn main() {
    let mut s = String::from("hello");

    change(&mut s); // 可变引用
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
```

在**同一时间**，只能有**一个**对某一特定数据的可变引用

这样的好处是能在编译阶段就避免**数据竞争**

数据竞争:
  - 两个或更多指针同时访问同一数据。
  - 至少有一个指针被用来写入数据。
  - 没有同步数据访问的机制。

我们也不能在拥有不可变引用的同时拥有可变引用

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s; // 没问题
    let r2 = &s; // 没问题
    let r3 = &mut s; // 大问题,编译阶段直接报错

    println!("{}, {}, and {}", r1, r2, r3);
}
```

一个引用的作用域从声明的地方开始一直持续到最后一次使用为止

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s; // 没问题
    let r2 = &s; // 没问题
    println!("{} and {}", r1, r2);
    // 此位置之后 r1 和 r2 不再使用

    let r3 = &mut s; // 没问题
    println!("{}", r3);
}

```

悬垂指针例子:

```rust
fn main() {
    let reference_to_nothing = dangle();
}

fn dangle() -> &String {
    let s = String::from("hello");

    &s // 返回s的引用
} // 在此处s的内存被释放,返回的指针指向了一片可能已经被分配的内存空间
```

引用总结:
  - 在任意给定时间,要么只能有一个可变引用,要么只能有多个不可变引用
  - 引用必须总是有效的

## 切片类型

另一个没有所有权的数据类型是 `slice` , `slice` 允许引用集合中一段连续的元素序列,而不用引用整个集合

当我们需要获取集合中的索引之类的数据时候,返回值为usize,但是返回值与集合之间没有任何关系,若集合发生变化,则数据直接失效

例:

```rust
fn first_word(s: &String) -> usize {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return i;
        }
    }

    s.len()
}

fn main() {
    let mut s = String::from("hello world");

    let word = first_word(&s); // word 的值为 5

    s.clear(); // 这清空了字符串，使其等于 ""

    // word 在此处的值仍然是 5，
    // 但是没有更多的字符串让我们可以有效地应用数值 5。word 的值现在完全无效！
}

```

### 字符串 slice

```rust
fn main() {
    let s = String::from("hello world");

    let hello = &s[0..5];
    let world = &s[6..11];
}
```

实际上类似于 `python` 的切片

因此 `&str` 返回的就是一个切片类型,它是一个指向二进制程序特定位置的 `slice`
