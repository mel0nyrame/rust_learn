# 枚举和模式匹配

## 定义枚举

```rust
#![allow(unused)]
fn main() {
    enum IpAddrKind {
        V4,
        V6,
    }
}
```

## 使用枚举

```rust
#![allow(unused)]
fn main() {
    enum IpAddrKind {
        V4,
        V6,
    }

    let four = IpAddrKind::V4;
    let six = IpAddrKind::V6;
}
```

枚举类型可以关联类型

```rust
#![allow(unused)]
fn main() {
    enum IpAddr {
        V4(String),
        V6(String),
    }

    let home = IpAddr::V4(String::from("127.0.0.1"));

    let loopback = IpAddr::V6(String::from("::1"));
}
```

也可以关联元组

```rust
#![allow(unused)]
fn main() {
    enum IpAddr {
        V4(u8, u8, u8, u8),
        V6(String),
    }

    let home = IpAddr::V4(127, 0, 0, 1);

    let loopback = IpAddr::V6(String::from("::1"));
}
```

枚举内可以嵌套枚举

可以在枚举上定义方法

## 另一个常用枚举`Option`

`Option` 是标准库定义的另一个枚举

编程语言的设计经常要考虑包含哪些功能,但考虑排除哪些功能也很重要

`Rust` 并没有很多其他语言中有的空值功能,空值( `Null` )是一个值,它代表**没有值**

空值的问题在于当你尝试像一个非空值那样使用一个空值,会出现某种形式的错误,因为空和非空的属性无处不在,非常容易出现这类错误

`Rust` 并没有空值,不过它确实拥有一个可以编码存在或不存在概念的枚举:`Option<T>`

```rust
#![allow(unused)]
fn main() {
    enum Option<T> {
        Some(T),
        None,
    }
}
```

`Option<T>` 和 `T` 是不同类型，编译器强制在使用前处理 `None` 的情况(硬约束,不处理空指针问题就编译不通过)

只要一个值不是 `Option<T>` 类型,就绝对不可能为空,可以安全使用

## match 控制流运算符

`match` 类似于其他语言的 `switch`，但是更强大

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}

fn main() {}

```

### 绑定值的模式

有点类似于 `lambda` 表达式

```rust
#[derive(Debug)] // 这样可以立刻看到州的名称
enum UsState {
    Alabama,
    Alaska,
    // --snip--
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
    // --snip--
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        // 在这里就可以获取到传入进来的 UsState 值
        Coin::Quarter(state) => {
            println!("State quarter from {:?}!", state);
            25
        }
    }
}

fn main() {
    value_in_cents(Coin::Quarter(UsState::Alaska));
}
```

### 匹配 Option<T>

在此前的枚举中可以使用 `Option<T>` 来避免空指针，若要使用 `Option<T>` 就可以使用 `match` 来匹配处理空指针问题

```rust
fn main() {
    fn plus_one(x: Option<i32>) -> Option<i32> {
        match x {
            // 若值为 None，则返回 None
            None => None,
            Some(i) => Some(i + 1),
        }
    }

    let five = Some(5);
    let six = plus_one(five);
    let none = plus_one(None);
}

```

`match` 的匹配是穷举式的，rust强制要求处理全部值或值为 `None` 的情况

### 通配模式和 _ 占位符

`match` 可以只对部分特殊值处理，其他值统一处理

```rust
fn main() {
    let dice_roll = 9;
    match dice_roll {
        3 => add_fancy_hat(),
        7 => remove_fancy_hat(),
        // 处理其他值
        other => move_player(other),
        // 不想使用值，rust编译器警告，也不想做任何实际功能
        // _ => (),
    }

    fn add_fancy_hat() {}
    fn remove_fancy_hat() {}
    fn move_player(num_spaces: u8) {}
}
```

## if let 简单控制流

为了结合 `if` 和 `let` 并简化代码，则可以使用 `if let`语法

```rust

#![allow(unused)]
fn main() {
    let some_u8_value = Some(0u8);

    // 与下面等价
    match some_u8_value {
        Some(3) => println!("three"),
        _ => (),
    }

    if let Some(3) = some_u8_value {
        println!("three");
    }
}
```

但是这样会失去 `match` 强制要求的穷尽性检查，让其他情况统一为 `_ => ()`，可以说是语法糖

`if let` 也可以使用 `else`

```rust

#![allow(unused)]
fn main() {
    #[derive(Debug)]
    enum UsState {
        Alabama,
        Alaska,
    }

    enum Coin {
        Penny,
        Nickel,
        Dime,
        Quarter(UsState),
    }
    let coin = Coin::Penny;


    // 上下等价
    let mut count = 0;
    match coin {
        Coin::Quarter(state) => println!("State quarter from {:?}!", state),
        _ => count += 1,
    }

    if let Coin::Quarter(state) = coin {
        println!("State quarter from {:?}!", state);
    } else {
        count += 1;
    }
}

```
